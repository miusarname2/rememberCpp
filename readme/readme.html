Visual C++ for Linux Development

Configuración del proyecto para Linux Development

Con esta carga de trabajo puede crear código de C++ para servidores, escritorios y dispositivos de Linux. Puede administrar las conexiones a estas máquinas desde VS. VS copiará automáticamente y creará remotamente sus orígenes, y puede iniciar la aplicación con el depurador. Nuestro sistema de proyectos admite arquitecturas específicas de destino, incluido ARM.

Conectando a Linux

Requisitos previos

Actualmente solo se admite la compilación remota en la máquina de destino de Linux. No estamos limitados por distribuciones de Linux específicas, pero dependemos de la presencia de algunas herramientas. En concreto, necesitamos openssh-server, g++, gdb y gdbserver. Use su administrador de paquetes favorito para instalarlas, por ejemplo, en sistemas basados en Debian: sudo apt-get install openssh-server g++ gdb gdbserver

Primera conexión

La primera vez que se dirija a un equipo Linux, se le solicitará la información de conexión. Esto se desencadena al compilar el proyecto.

Agregar y quitar conexiones

Para agregar una conexión nueva, vaya a Herramientas > Opciones y busque Conexión; Connection Manager estará bajo Multiplataforma. Desde aquí puede agregar y quitar conexiones.

Para cambiar la conexión que usa un proyecto, vaya a la configuración general de propiedades del proyecto y actualice la opción Máquina de compilación remota.

Propiedades del proyecto

Todas las opciones necesarias para controlar la compilación de C++ se exponen en las páginas de propiedades del proyecto. Describiremos algunos conceptos específicos sobre el funcionamiento de Linux. En primer lugar, bajo configuración general, verá que la raíz remota está establecida en ~/projects/ de forma predeterminada y que estamos configurando el directorio del proyecto remoto para que coincida con el nombre de nuestro proyecto en esa ubicación.

Observando la configuración general del proyecto, puede ver cómo se configuraron nuestros directorios de salida e intermedios. Además, verá que este proyecto se configuró como una aplicación; por tanto, nuestro archivo ejecutable está en bin/x64/Debug/ como ConsoleApplication1.out. Tenga en cuenta que en los tipos de configuración también se admiten bibliotecas estáticas y dinámicas.

Agregue dependencias de biblioteca adicionales en la página Enlazador > Propiedad de entrada.

Puede pasar comandos adicionales anteriores al inicio al depurador para realizar acciones como iniciar aplicaciones gráficas en el equipo Linux remoto.

También puede enviar eventos posteriores a la compilación para controlar el comportamiento remoto, como en este ejemplo, en el que se exporta un PIN de GPIO para su uso sin necesidad de ejecutar el elemento ejecutable como superusuario.
